РАБОТА С ФАЙЛАМИ
git status - показывает какие файлы изменены и готовы к коммиту
git add . - добавляет все изменения для коммита
git add <название файла> - добавляет файл к следующему коммиту
git commit -m "Что было добавлено удалено и.т.д" - сохраняет изменения

ОТПРАВКА ИЗМЕНЕНИЙ
git push origin <ветка> - отправляет изменения в текущей ветке
git push origin --delete <ветка> - удаляет ветку с сервера

ПОЛУЧЕНИЕ ИЗМЕНЕНИЙ
git pull - получает изменения с сервера и сразу объединяет их с твоими
git fetch - только скачивает изменения (без объединения). Полезно для проверки
git merge origin/<ветка> - объединяет удаленную ветку с текущей (после fetch)

РАБОТА С ВЕТКАМИ
git branch - показывает локальные ветки 
git branch <ветка> - создает ветку
git checkout <ветка> - переключается на существующую ветку
git checkout -b <ветка> - создает ветку и сразу на нее переключается
git merge <ветка> - вливать ветку в текущую
git push -u origin <ветка> - отправляет новую ветку на сервер и связывает с удаленной

ОТМЕНА И ОТКАТ
git reset HEAD~1 - удаляет последний коммит но оставляет изменения в файлах
git reset --hard HEAD~1 - полный откат: удаляет коммит и изменения
git checkout -- <файл> - возвращает файл в состояние последнего коммита (удаляет изменения)
git restore --staged <файл> - убирает файл из добавленных к коммиту (git add), но не удаляет изменения
